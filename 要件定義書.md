# HMEP レクチャー動画管理パイプライン
## システム要件定義書

ハワイ大学沖縄事務所 / 作成日：2026年5月8日

---

## 1. プロジェクト概要

本プロジェクトは、HMEP（Hawaii Medical Education Program）で実施されるレクチャーの情報管理・動画公開プロセスを自動化するパイプラインを Claude Code を用いて構築するものです。現在、①メール確認、②動画リネーム、③アップロードをそれぞれ手作業で行っており、作業ミス・工数増大のリスクがあります。本システムはこれら三工程を Python スクリプト群として実装し、ワンコマンドで一括処理できる環境を整備します。

**【対象ユーザー】**
- ハワイ大学沖縄事務所スタッフ（主に瑞慶覧さん：研修評価・コアレク担当）
- 事務長（最終確認・承認役）

**【解決する課題】**
- メールからのレクチャー情報の転記ミス防止
- 動画ファイルの命名規則の統一
- YouTube アップロード・説明文設定の工数削減
- レクチャー記録の一元管理（Excel 台帳化）

---

## 2. システム構成

本パイプラインは以下の 3 つのサブシステムで構成されます。

| 番号 | サブシステム | 主な機能 | 実装言語/ツール |
|------|------------|---------|--------------|
| ① | メール解析・レクチャー表生成 | Outlookメールからレクチャー情報を抽出しExcel/CSV出力 | Python (win32com, openpyxl) |
| ② | 動画リネーム | レクチャー表に基づき動画ファイルを一括リネーム | Python (pathlib, os) |
| ③ | YouTubeアップロード | リネーム済み動画をYouTubeへアップロード・説明文設定 | Python (YouTube Data API v3) |

> ※ 各サブシステムは独立して実行可能。`main.py` から一括実行も可能。  
> ※ 実行環境：Windows 10/11 + Python 3.10 以上（Claude Code 対応環境）

## 3. 詳細要件

### 3.1　① メール解析・レクチャー表生成

#### 3.1.1 入力
- Outlook の特定フォルダ（例：「HMEPレクチャー」）内のメール
- 取得方法：`win32com.client`（Windows Outlook API）経由
- 対象期間：設定ファイルで指定（例：2026年度分すべて）

#### 3.1.2 処理
- 件名・本文から正規表現でレクチャー情報をパース
- 対象フォーマット：【全学年対象】【タイトル】【講師】【講義紹介文】【講師略歴】の各ブロック
- 解析結果を構造化データとして保持
- 既存レクチャー表との差分チェック（重複登録防止）

#### 3.1.3 出力

`lectures.xlsx` に以下のフィールドを列として出力：

| フィールド名 | 内容 | 例 |
|------------|------|-----|
| `lecture_date` | レクチャー日（日付） | 2026-05-09 |
| `target_grade` | 対象学年 | 全学年対象 |
| `title` | レクチャータイトル | Global HealthとPlanetary Health… |
| `speaker_name` | 講師氏名 | 坂口 公太 |
| `speaker_affiliation` | 所属機関・役職 | 島根大学医学部附属病院 総合診療医センター助教 |
| `description` | 講義紹介文 | 前半は…後半は… |
| `biography` | 講師略歴 | 坂口 公太 先生（Kota Sakaguchi…） |
| `email_subject` | 元メール件名 | [HMEP] 2026/5/9 レクチャーのご案内 |
| `email_received_at` | メール受信日時 | 2026-05-01 09:00 |

- 解析不能だったメールのリストを別シートに出力（要手動確認）

> ※ メール本文フォーマットが変更された場合、`config.yaml` の正規表現パターンを更新する運用とする。

---

### 3.2　② 動画リネーム

#### 3.2.1 入力
- 動画フォルダパス（`config.yaml` で設定）
- `lectures.xlsx`（①の出力）
- 対応付け方法：ファイルの更新日時 ± 24 時間 + ファイル名の類似度でマッチング

#### 3.2.2 ファイル命名規則

| 要素 | 形式 | 例 |
|------|------|-----|
| 日付 | `YYYY M D` | 2026 5 9 |
| 講師名 | 姓 名 + 先生 | 坂口 公太 先生 |
| タイトル | ダブルクォートで囲む | `"Global Health と Planetary Health"` |
| 拡張子 | 元ファイルを保持 | `.mp4` |

**完成例：**
```
2026 5 9 坂口 公太 先生 "Global Health と Planetary Health".mp4
```

- 元ファイルは `videos_renamed/` フォルダへコピー（元ファイルを削除しない）
- ファイル名に使用できない文字（`\ / : * ? " < > |`）は自動で除去またはスペースに置換

#### 3.2.3 出力
- リネーム済みファイル（`videos_renamed/` フォルダ）
- リネームログ（元ファイル名 → 新ファイル名の対応表）を `lectures.xlsx` に追記

---

### 3.3　③ YouTube アップロード

#### 3.3.1 認証
- YouTube Data API v3 を使用
- Google OAuth 2.0 認証（初回のみブラウザ認証 → トークンをローカル保存）
- `credentials/` フォルダに `client_secrets.json` と `token.json` を格納（`.gitignore` 必須）

#### 3.3.2 入力
- `videos_renamed/` フォルダ内の動画ファイル
- `lectures.xlsx`（②の出力、説明文生成用）

#### 3.3.3 YouTube 設定項目

| 設定項目 | 設定値 | 備考 |
|--------|--------|------|
| タイトル | ファイル名から生成（拡張子除く） | リネーム後のファイル名を流用 |
| 説明文 | レクチャー情報全文（テンプレート） | 下記テンプレート参照 |
| 公開設定 | 限定公開（unlisted）デフォルト | 管理者が後から変更可能 |
| プレイリスト | 年度別プレイリストへ自動追加 | 例：HMEP Lectures 2026 |
| タグ | HMEP, 講師名, 年度 | 検索最適化 |
| 言語 | ja（日本語） | メタデータに設定 |

#### 3.3.4 説明文テンプレート

```
【レクチャー情報】
開催日：{lecture_date}
対象：{target_grade}
タイトル：{title}

【講師】
{speaker_name} 先生
{speaker_affiliation}

【講義紹介】
{description}

【講師略歴】
{biography}

── Hawaii Medical Education Program (HMEP) ──
ハワイ大学沖縄事務所
```

#### 3.3.5 出力
- アップロード済み動画の YouTube URL を `upload_log.csv` および `lectures.xlsx` に記録
- アップロード失敗分はエラーログに記録し再実行可能な状態を保持

## 4. ファイル構成

```
hmep_pipeline/
├─ 01_extract/
│   ├─ extract_lectures.py      # Outlookメール取得・解析スクリプト
│   └─ config.yaml              # フォルダ名・キーワード等の設定
├─ 02_rename/
│   └─ rename_videos.py         # レクチャー表と動画を突合・リネーム
├─ 03_upload/
│   ├─ upload_youtube.py        # YouTube Data API 連携スクリプト
│   └─ description_template.txt # 説明文テンプレート
├─ data/
│   ├─ lectures.xlsx            # 生成されたレクチャー表
│   └─ upload_log.csv           # アップロード履歴
├─ videos/                      # 元動画フォルダ（設定で変更可）
├─ videos_renamed/              # リネーム後動画（元ファイル保持）
├─ credentials/                 # Google OAuth トークン（.gitignore対象）
└─ main.py                      # 全工程を一括実行するエントリーポイント
```

> ※ `credentials/` は `.gitignore` に追加し、Git リポジトリへの誤コミットを防ぐこと。

## 5. エラーハンドリング方針

| エラー種別 | 想定ケース | 対応方針 |
|----------|----------|---------|
| 解析失敗 | メール本文のフォーマットが異なる | スキップして警告ログ出力・手動確認リストに追加 |
| 動画未検出 | 対応する動画ファイルが見つからない | レクチャー表に「動画なし」フラグを立てて継続 |
| API上限 | YouTube APIのクォータ超過 | アップロードを中断し翌日分をキューに保持 |
| 認証エラー | OAuthトークン期限切れ | 自動リフレッシュ・失敗時に再認証ガイドを表示 |
| 重複アップ | 同一レクチャーの二重登録 | `upload_log.csv` で既存チェック・スキップ |

> ※ 全エラーはタイムスタンプ付きで `logs/pipeline_YYYY-MM-DD.log` に記録する。

## 6. 制約・前提条件

- 実行環境：Windows 10/11、Python 3.10+、Outlook デスクトップアプリ（Microsoft 365）がインストールされていること
- Outlook はスクリプト実行時に起動済みであること（win32com 要件）
- YouTube アカウント：ハワイ大学沖縄事務所の Google アカウントに紐づけた YouTube チャンネルを使用
- Google Cloud Console にてプロジェクト作成・YouTube Data API v3 有効化・OAuth クライアント ID 発行が事前に必要
- YouTube API 無料枠クォータ：1 日 10,000 ユニット（動画アップロード 1 本 = 1,600 ユニット消費、1 日最大約 6 本）
- 動画フォーマット：MP4 推奨（MOV, AVI 等も対応予定）
- レクチャー 1 件につき動画 1 ファイルを前提とする（複数分割は対象外）

> ※ クォータ超過時は翌日に自動で続きからアップロード再開する設計とする。
